
Each agent has its own soul and episodic memory, completely its own.
ALSO
During its sleep cycle, when it's contributing its daily memories to long-term memory
it ALSO contributes its most salient memories to a hierarchical memory store one level up.

So, projects may comprise numerous agents, and there is a global project memory.
Projects would not contribute across to other projects at the same level in the hierarchy, and this prevents bleed between projects.

Users have their own global memory that is the top level of their memory hierarchy, available for all projects they're on.
---
We start simple: episodic memory with precompaction dumps and post-conversation notation with daily roundups going to long-term storage + semantic search as a tool for the agent. I.e. the OpenClaw memory style.

Once we have that, we build out the agents/projects/users hierarchical version of it.

Then we picks/build some evals and improve aspects of memory (e.g. search) over time to improve our benchmarks.

One question comes up - how to deal with full organizational memory?
I think this amounts to: an organization is a set of projects, and the access someone has to an organization is directly correlated with which projects they're on. We don't need a separate "organizational" memory. To achieve something like that beyond what the projects already give us maybe we would create a meta "project" for our own organzation.
